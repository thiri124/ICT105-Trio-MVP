# Selected Semester Project Opportunity

## Selected Project Title

Campus Item Lending & Skill Sharing Platform

## Target User

University students and campus club members who need temporary access to academic tools, equipment, study materials, or peer assistance.

### Who has the problem?

University students and campus club members.

## Problem Statement

University students frequently need short-term access to specialized tools, textbooks, equipment, or peer assistance, but finding available resources on campus is often difficult, time-consuming, and unorganized.

### What problem does the user experience?

Students often spend significant time searching through Facebook groups, Discord servers, and personal networks to find items or assistance. Information is scattered, requests are easily missed, and there is no centralized platform for resource sharing.

## Current Alternative

Students currently rely on Facebook groups, Discord servers, friends, and personal networks to request items or assistance.

### How do users currently solve or manage this problem?

They post requests in social media groups, ask friends directly, or contact club members. These methods are informal and often inefficient.

## Proposed IT Solution

A web-based campus platform where students can list items they are willing to lend, offer skills they can provide, and submit requests for resources they need. The platform will include user accounts, listings, and a simple request and approval workflow.

### What app, system, dashboard, or platform will the team develop?

A Campus Item Lending & Skill Sharing Web Application.

## Why This Opportunity Was Selected

The idea received the highest NUF score because it is highly useful, relatively unique, and technically feasible for a semester-long project. It solves a real problem experienced by many students and provides clear value to the campus community.

The project achieved the highest score because it addresses a genuine student need while remaining realistic to implement using standard web technologies.

## Why This Is Feasible Without IoT or Advanced Cybersecurity

The project only requires a standard web application with authentication, database management, and request tracking features. No hardware devices, sensors, IoT components, or advanced cybersecurity systems are required.

### Explain why the team can build or simulate the MVP using available tools.

The MVP can be developed using React, Tailwind CSS, and Supabase. These technologies provide authentication, database functionality, and hosting capabilities that are sufficient for a working prototype.
